# Verification Rules

After completing work, verify it at increasing levels of confidence.
Do not treat "asset exists" or "compile succeeded" as proof the work is done.

## 1. Structure Proof
Confirm you edited the correct asset. Inspect the editor context, list the
Blueprint graph or widget tree, verify asset identity before moving on.

## 2. Compile Proof
Verify technical health: Blueprint compiles, material compiles, no errors.
This is the minimum gate, not the quality bar.

## 3. In-World Proof
Verify it looks correct in context. Take a screenshot, observe the scene,
check placement and visual quality at gameplay framing distance.

## 4. Runtime Proof
When the task involves behavior (gameplay, animation, UI interaction),
start PIE and verify it works during actual play. Visual-only changes
can stop at level 3.
